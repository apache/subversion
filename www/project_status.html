Project Status

Subversion Releases

Releases go through a stabilization phase as described in the HACKING file, see the section "Stabilizing and maintaining releases". The CHANGES file contains a short description of what's new in each release. Releases are always source tarballs; binary package updates for various OS's generally follow the source tarball by a few days.

For long-term plans, please see the roadmap page.

Pre-1.0 releases that included a change in the filesystem database schema (and therefore required a repository reload), are highlighted in red. There will be no more such releases before Subversion 2.0.
